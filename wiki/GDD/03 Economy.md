## 5. Экономика: ресурсы и здания

### 5.1. Архитектура поселения

Места для строительства **строго захардкожены** на сцене.

Игрок **не** выбирает *где* строить — слоты уже расставлены. Он выбирает *когда* вкладывать ресурсы в открытый слот.

Отдельного меню / сайдбара строительства **нет**.

### 5.2. Строительство (оффер на слоте)

Стройка работает **как оффер Пирамиды + таймер крафта дома**:

1. Слот на сцене изначально — **руины** (Locked): без HUD, DnD не принимает, молния не бьёт.
2. Фаза **разблокирует** здание(я) → слот становится **Buildable**: появляется World HUD оффера стройки.
3. Игрок DnD-ит ресурсы в слот по позициям оффера (несколько типов, как у Пирамиды).
4. Оффер закрыт → включается **таймер/слайдер** (как produce HUD) на `buildDurationSeconds`.
5. Таймер дошёл → слот **Built**: обычное производство, workers/input, молния может бить.

#### Состояния слота

| Состояние | Визуал | HUD | DnD | Молния |
| --- | --- | --- | --- | --- |
| **Locked** | руины | нет | нет | нет |
| **Buildable** | руины / недострой | оффер стройки (`n/m` по позициям) | только ресурсы из cost | нет |
| **Constructing** | «в процессе» | прогресс-бар таймера | нет (оффер уже закрыт) | нет |
| **Built** | готовое здание | production HUD | input + жители | да |

#### Данные: `BuildingDefinition`

Стоимость и длительность стройки живут на **`BuildingDefinition`** (SO), не в фазе:

| Поле | Смысл |
| --- | --- |
| `buildCost[]` | позиции оффера: `ResourceDefinition` + `count` (как у Пирамиды, без `secondsReward`) |
| `buildDurationSeconds` | длительность таймера после закрытия оффера |
| production… | input/output/workers/duration — как сейчас у `BuildingRecipe` (или ссылка на recipe) |

Фаза только **открывает** здания: `unlockBuildingIds[]` в `PhaseDefinition` (см. [[04 Timeline & Events]]).

#### Прогрессия

- В начале рана Built только стартовые здания (phase 0 / loadout).
- Остальные слоты — Locked-руины без HUD.
- Новые здания открываются **по фазам**, чтобы не вываливать весь интерфейс сразу.
- Пример: фаза 3 → unlock трёх зданий → у каждого свой cost из его `BuildingDefinition`.

#### Вне MVP

Апгрейды зданий (скорость, новые outputs) — не в этом заходе. Меню строительства не планируется.

### 5.3. Таблица производственных цепочек

| Здание | Требования для работы (Input) | Результат (Output) | Описание |
| --- | --- | --- | --- |
| Дом Предков | Нет (генерирует пассивно) | Пшеница (Еда) | Стартовое здание. Сердце поселения. |
| Дом Предков (Размножение) | Пшеница (+ рабочие в здании) | Карточка жителя | Создание рабочей силы в лоток Villager. |
| Хижина дровосека | рабочие (карты жителей на здании) | Дерево | Базовый ресурс. |
| Каменоломня | рабочие | Камень | Базовый ресурс. |
| Мастерская | рабочие + Дерево + Камень | Обсидиан (Идолы) | Сложный ресурс 2-го порядка для продвинутых требований богов. |
| Храм Шамана | рабочие | Карточки-Активки | Генерирует баффы для зданий (шанс дропа). |
| Алтарь (в Храме) | 1 рабочий (карта жителя уничтожается) | Кровь | Элитный ресурс для финальных этапов игры. |

> Жители — тоже карточки: назначение на здание через DnD, `[-]` возвращает карту в лоток. Input/output зданий — ссылки на `ResourceDefinition`. См. [[02 Entities]] · [[../Architecture/06 Inventory|Inventory]].

> **Примечание:** любое здание при выдаче ресурса имеет небольшой шанс дополнительно выдать [[02 Entities#Активки (Баффы)|Карточку-Активку]].
